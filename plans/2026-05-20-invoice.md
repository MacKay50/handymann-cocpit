# Plan — Invoice (Faktura)

**Slug:** 2026-05-20-invoice  
**Date:** 2026-05-20  
**Classification:** Standard

## Vision invariants
- §2 LLM recommends, code decides → subtotal/vat/total server-computed ✓
- §3 No silent 0-defaults → financial fields have no default in table model ✓
- §4 Append-only for financial records → soft-delete; sent/paid/cancelled are immutable ✓
- §5 Flat structure → registered same way ✓
- company_id + customer_id server-derived from project ✓

## Out of scope
- Automatic import of TimeEntries/Expenses as invoice lines (FU-12)
- PDF generation / email sending
- e-conomic export format (separate future feature)
- Payment records against invoices (FU-3: Payment resource)
- Approval flow

## Phase 1 — Invoice model (TDD)

**New:** `src/haandvaerker/models/invoice.py`

```
InvoiceStatus(str, Enum): draft | sent | paid | cancelled
InvoiceSequence(table): year(PK int), last_number int

InvoiceLineCreate(SQLModel):
    description: str = Field(min_length=1, max_length=500)
    unit: Optional[str] = Field(default=None, max_length=50)
    quantity: float = Field(gt=0)
    unit_price: float = Field(ge=0)

InvoiceLine(SQLModel, table=True):
    id, invoice_id(FK), description, unit, quantity, unit_price, line_total

InvoiceLineRead(SQLModel):
    id, description, unit, quantity, unit_price, line_total

InvoiceCreate(SQLModel):
    id: Optional[str] = None
    project_id: str
    title: str = Field(min_length=1, max_length=200)
    description: Optional[str] = None
    issue_date: date
    due_date: date
    notes: Optional[str] = None
    lines: list[InvoiceLineCreate] = []

Invoice(SQLModel, table=True):
    id, company_id(FK), project_id(FK), customer_id(str),
    invoice_number(unique), title, description, issue_date, due_date, notes,
    status(default=draft), subtotal, vat_amount, total (no defaults),
    active(default=True)

InvoiceRead(SQLModel):
    id, company_id, project_id, customer_id,
    invoice_number, title, description, issue_date, due_date, notes,
    status, subtotal, vat_amount, total, active,
    lines: list[InvoiceLineRead] = []

InvoiceUpdate(SQLModel):
    title, description, issue_date, due_date, notes, lines (replaces all)

InvoiceSummary(SQLModel):
    project_id, total_invoiced, total_paid, outstanding

VAT_RATE = Decimal("0.25")
compute_line_total(quantity, unit_price) → float  (Decimal ROUND_HALF_UP)
compute_invoice_totals(line_totals) → tuple[float, float, float]
```

### Acceptance criteria
- All model classes importable
- `compute_invoice_totals([1000.0]) == (1000.0, 250.0, 1250.0)`

## Phase 2 — Invoice API + tests (TDD)

**New:** `src/haandvaerker/api/invoices.py`, `tests/test_invoices.py`  
**Modified:** `src/haandvaerker/main.py`, `src/haandvaerker/models/__init__.py`

### Endpoints

| Method | Path | Status | Notes |
|--------|------|--------|-------|
| POST | /invoices/ | 201 | validate project; derive company_id, customer_id; compute totals |
| GET | /invoices/summary | 200 | project_id required; BEFORE /{id} |
| GET | /invoices/ | 200 | active_only, company_id?, project_id?, status? |
| GET | /invoices/{id} | 200 | includes inactive |
| PATCH | /invoices/{id} | 200 | 409 if not draft |
| DELETE | /invoices/{id} | 204 | soft-delete |
| POST | /invoices/{id}/send | 200 | draft → sent |
| POST | /invoices/{id}/pay | 200 | sent → paid |
| POST | /invoices/{id}/cancel | 200 | draft/sent → cancelled |

VALID_TRANSITIONS = {draft: {sent, cancelled}, sent: {paid, cancelled}}

### Acceptance criteria
1. POST returns invoice_number `FKT-2026-001`; second invoice `FKT-2026-002`
2. company_id == project.company_id; customer_id == project.customer_id (not in create payload)
3. Line totals, subtotal, vat_amount, total are all server-computed
4. due_date < issue_date → 422
5. Unknown project → 422; inactive project → 422
6. GET /invoices/?project_id=X filters correctly
7. GET /invoices/summary returns correct total_invoiced, total_paid, outstanding
8. GET /invoices/summary?project_id=ukendt → 422
9. PATCH on draft updates + recalculates; PATCH on sent → 409
10. /send transitions draft→sent; /pay transitions sent→paid; /cancel from draft and sent
11. Invalid transition (pay from draft) → 409
12. DELETE soft-deletes; direct GET shows active=False

### Rollback
Delete `tests/test_invoices.py`, `api/invoices.py`, revert `main.py` and `__init__.py`.
