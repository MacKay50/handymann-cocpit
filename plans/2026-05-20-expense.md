## Phase 3 — Expense API + tests

**Files changed:**
- `src/haandvaerker/api/expenses.py` (new)
- `src/haandvaerker/main.py` (add router)
- `tests/test_expenses.py` (new — written first, TDD)

### Endpoints

| Method | Path | Status | Notes |
|--------|------|--------|-------|
| POST | /expenses/ | 201 | validates FK, computes amounts |
| GET | /expenses/ | 200 | active_only=True, project_id?, employee_id? |
| GET | /expenses/summary | 200 | project_id required; BEFORE /{id} |
| GET | /expenses/{id} | 200 | includes inactive |
| PATCH | /expenses/{id} | 200 | date/description/receipt_ref/billable only |
| DELETE | /expenses/{id} | 204 | soft-delete |

### Acceptance criteria

1. `POST /expenses/` with `category=materialer`, `amount_excl_vat=1000` returns `vat_amount=250`, `amount_total=1250`
2. `POST /expenses/` with `category=transport_km`, `km=100` returns `amount_excl_vat=376`, `vat_amount=0`, `amount_total=376`
3. `POST /expenses/` with `category=transport_km`, `km=10`, `km_rate=4.0` returns `amount_excl_vat=40`, `amount_total=40`
4. `POST /expenses/` with `category=parkering`, `amount_excl_vat=50` returns `vat_amount=0`, `amount_total=50`
5. `POST /expenses/` with `category=parkering`, `amount_excl_vat=50`, `apply_vat=True` returns `vat_amount=12.5`, `amount_total=62.5`
6. Unknown project → 422
7. Inactive project → 422
8. Unknown employee → 422
9. Inactive employee → 422
10. `GET /expenses/?project_id=<id>` returns only matching entries
11. `GET /expenses/summary?project_id=<id>` returns correct totals
12. `GET /expenses/summary?project_id=ukendt` → 422
13. `PATCH /expenses/{id}` updates description/billable
14. `DELETE /expenses/{id}` → 204, entry excluded from list, direct GET shows active=False

### Rollback

Delete `tests/test_expenses.py`, delete `src/haandvaerker/api/expenses.py`,
revert `main.py`.
