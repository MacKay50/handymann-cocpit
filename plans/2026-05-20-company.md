# Plan — Company (Virksomhed) + Multi-firma retrofit

**Slug:** 2026-05-20-company  
**Date:** 2026-05-20  
**Classification:** Standard

## Vision invariants
- §1 Kunden er omdrejningspunktet → Company sits above Customer, not replacing it ✓
- §3 Ingen stille defaults → company_id required on Customer/Employee, never defaulted ✓
- §5 Flad struktur → Company registered same way as all other resources ✓
- §6 CVR maskeres → CompanyRead exposes cvr_masked, not cvr_number ✓

## Out of scope
- Company-level settings or branding
- Per-company user roles/auth
- Cross-company reporting

## Phase 1 — Company model + API + tests (TDD)

**New files:**
- `src/haandvaerker/models/company.py`
- `src/haandvaerker/api/companies.py`
- `tests/test_companies.py`

**Modified:**
- `src/haandvaerker/main.py`
- `src/haandvaerker/models/__init__.py`

### Company model
```
Company(table=True): id(UUID), name, cvr_number, address, phone, email, logo_ref, active
CompanyCreate: id=None, name(required), cvr_number?, address?, phone?, email?, logo_ref?
CompanyRead: id, name, cvr_masked, address, phone, email, logo_ref, active
CompanyUpdate: name?, address?, phone?, email?, logo_ref?
```

### Endpoints
POST /companies/ 201 | GET /companies/ | GET /companies/{id} | PATCH /companies/{id} | DELETE /companies/{id} 204

### Acceptance criteria (Phase 1)
1. POST /companies/ creates company, returns cvr_masked not cvr_number
2. GET /companies/ active_only=True default
3. DELETE soft-deletes; direct GET shows active=False
4. POST with duplicate id → 409
5. Inactive company can still be fetched by id

### Rollback
Delete `models/company.py`, `api/companies.py`, `tests/test_companies.py`, revert `main.py` and `__init__.py`.

## Phase 2 — Retrofit company_id onto all existing resources

**Modified models:**
- `models/customer.py` — add company_id FK to Customer table; add to CustomerCreate; add to CustomerRead/from_orm_masked
- `models/employee.py` — same pattern
- `models/project.py` — add company_id to Project table + ProjectRead (NOT ProjectCreate — derived)
- `models/quote.py` — add company_id to Quote table + QuoteRead
- `models/time_entry.py` — add company_id to TimeEntry table + TimeEntryRead
- `models/expense.py` — add company_id to Expense table + ExpenseRead

**Modified APIs:**
- `api/customers.py` — add _require_active_company; add company_id filter to list
- `api/employees.py` — add _require_active_company; add company_id filter to list
- `api/projects.py` — derive company_id from customer; add company_id filter to list
- `api/quotes.py` — derive company_id from project; add company_id filter to list
- `api/time_entries.py` — derive company_id from project; add company_id filter to list
- `api/expenses.py` — derive company_id from project; add company_id filter to list

**Modified tests:**
- `tests/conftest.py` — add company_id fixture
- `tests/test_customers.py` — each test: add company_id param + pass to creates
- `tests/test_employees.py` — same
- `tests/test_projects.py` — update _make_customer to create company first
- `tests/test_quotes.py` — update _make_customer to create company first
- `tests/test_time_entries.py` — update _setup to create company first
- `tests/test_expenses.py` — update _setup to create company first

### Acceptance criteria (Phase 2)
1. POST /customers/ without company_id → 422
2. POST /customers/ with unknown company_id → 422
3. POST /customers/ with inactive company_id → 422
4. GET /customers/?company_id=X returns only customers for that company
5. Project.company_id == Customer.company_id (server-derived, client cannot override)
6. GET /projects/?company_id=X filters correctly
7. TimeEntry.company_id matches its Project.company_id
8. All 90+ prior tests pass (company_id threaded through helpers)

### Rollback
Revert model and API changes; revert test files to pre-phase state.
